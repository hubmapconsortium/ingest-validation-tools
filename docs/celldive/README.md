## Directory structure

| pattern | required? | description |
| --- | --- | --- |
| `channel_list\.txt` | ‚úì | Information about the capture channels and tags (comma separated) |
| `slide_list\.txt` | ‚úì | Information about the slides used by the experiment- each line corresponds to a slide name (begins with S - e.g. S20030077) - used in filenames |
| `HuBMAP_OME/region_[^/]+/S[^/]+\.ome\.tif` | ‚úì | OME TIFF Files for the corresponding region (e.g. region_001) by slide (e.g S20030077) |
| `HuBMAP_rounds/round_info_S[^/]*\.dat` | ‚úì | metadata file for the capture by slide (e.g S20030077) item-value tab separated format |
| `HuBMAP_Seg_and_quant/gray_scale_T_cells/region[^/]*/mask_T_cells_slideS[^/]+\.tif` | ‚úì | grayscale T-cell masks |
| `HuBMAP_Seg_and_quant/quantification/region[^/]*/quant_slideS[^/]+\.csv` | ‚úì | Comma separated quantification files |
| `HuBMAP_Seg_and_quant/rgb_T_cells/region[^/]*/mask_T_cells_slideS[^/]+\.tif` | ‚úì | rgb T-cell masks |
| `HuBMAP_Seg_and_quant/segmentation/region[^/]*/dapi_slide_slideS[^/]+\.tif` | ‚úì | segmentation (dapi slides) |
| `vHE/S[^/]+\.ome\.tif` | ‚úì | vHE slides |
| `HuBMAP_OME/move_images.bat` |  | moves files |
| `HuBMAP_Seg_and_quant/*/move_images.bat` |  | moves files |
| `HuBMAP_OME/make_folders.bat` |  | creates directories |
| `HuBMAP_Seg_and_quant/*/make_folders.bat` |  | creates directories |
| `extras/.*` |  | Free-form descriptive information supplied by the TMC |
| `extras/thumbnail\.(png\|jpg)` |  | Optional thumbnail image which may be shown in search interface |

## Provenance

### `donor_id`
HuBMAP Display ID of the donor of the assayed tissue. Example: `ABC123`.

| constraint | value |
| --- | --- |
| pattern (regular expression) | `[A-Z]+[0-9]+` |
| required | `True` |

### `tissue_id`
HuBMAP Display ID of the assayed tissue. Example: `ABC123-BL-1-2-3_456`.

| constraint | value |
| --- | --- |
| pattern (regular expression) | `([A-Z]+[0-9]+)-[A-Z]{2}\d*(-\d+)+(_\d+)?` |
| required | `True` |

## Level 1

### `execution_datetime`
Start date and time of assay, typically a date-time stamped folder generated by the acquisition instrument. YYYY-MM-DD hh:mm, where YYYY is the year, MM is the month with leading 0s, and DD is the day with leading 0s, hh is the hour with leading zeros, mm are the minutes with leading zeros.

| constraint | value |
| --- | --- |
| type | `datetime` |
| format | `%Y-%m-%d %H:%M` |
| required | `True` |

### `protocols_io_doi`
DOI for protocols.io referring to the protocol for this assay.

| constraint | value |
| --- | --- |
| required | `True` |
| pattern (regular expression) | `10\.17504/.*` |

### `operator`
Name of the person responsible for executing the assay.

| constraint | value |
| --- | --- |
| required | `True` |

Email address for the operator.

| constraint | value |
| --- | --- |
| format | `email` |
| required | `True` |

### `pi`
Name of the principal investigator responsible for the data.

| constraint | value |
| --- | --- |
| required | `True` |

### `pi_email`
Email address for the principal investigator.

| constraint | value |
| --- | --- |
| format | `email` |
| required | `True` |

### `assay_category`
Each assay is placed into one of the following 3 general categories: generation of images of microscopic entities, identification & quantitation of molecules by mass spectrometry, and determination of nucleotide sequence.

| constraint | value |
| --- | --- |
| enum | `imaging` |
| required | `True` |

The specific type of assay being executed.

| constraint | value |
| --- | --- |

| constraint | value |
| --- | --- |
| type | `boolean` |
| required | `True` |
